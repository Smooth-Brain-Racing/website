How To Export ParaView Animations To Blender

If you want to render your CFD results, but your results exported from ParaView look like a Play Station 2 game rather than a super sick professional rendering, you'll need to use Blender. This is quick and easy for single timesteps, but ParaView only allows you export a scene from a single timestep. Blender also only allows you to import one scene at a time. Luckily there are python scripts that can be used to automate the process. Flexi CFD has Blender Pipeline (https://github.com/flexi-framework/BlenderPipeline) scripts that do this, but Blender changed there API since its release. I also found it to be more convenient to use a ParaView macro to export all of the x3d files instead of using the first script.

Simulation results post processed as they will be rendered

Prepare your results as usual. Apply whatever filters and colormaps you want rendered. Be sure to hide anything you don't want shown in the render including the lines, planes, and spheres used to set up streamlines, clips, and slices. Volume renders and Nvidia Index cannot be exported as x3d.

Importing Macro

Download the ParaView macro: link
Macros >> import new macro... 

ParaView Script Editor

Some of the parameters need to be edited in the macro.
Macros >> edit... >> exportx3d
Change D:/outputLocation/ to the path of the folder you want your x3d files to be saved in.
Change fileName to whatever you want your files to be called. They will be automatically numbered so they can be opened in order.
Save and close the Script Editor.

To run the macro go to Macros >> exportx3d. The output folder should start populating with x3d files. It could take some time and a lot of storage depending on how many timesteps there are.

The next step is to prepare your scene in Blender. Open blender and go to File >> Import >> X3D Extensible 3D (.x3d/.wrl). Select any of the timesteps from your output file.

x3d imported into Blender

You can delete the items highlighted above because Blender has a light and camera in the scene by default. Here you can set up the scene however you like by adding lights, adjusting camera, etc... It's easiest if you can avoid moving the imported Shape_IndexedFaceSet because you will need to enter the displacement into the animate_blender.py script later. This is because the script works by deletes and reimports the object for every frame.

In order to get the colormap data from ParaView to be visible, we need to create a material for our object.

Blender Shader Editor and Material Preview mode

To enter Material Preview mode click on the highlighted icon in the top right corner. This will make materials we create and assign visible in the Viewport. To edit materials, click on the highlighted icon in the bottom left and select Shader Editor. Then, with the object selected, click new.

Material Created

First, delete the Principled BSDF. Then Add >> Shader>> Diffuse BSDF. Add >> Input >> Attribute. In the object data properties on the right, we can find the active vertex color index on our object. Use the name of this for the name field in the attribute panel (It's usually Col by default). Connect The modules as shown and the colormap from Paraview should be visible on our object.

Complete Scene ready to be rendered

Select the shield icon next to the material name to make sure the material is not deleted when it is unassigned. Select the Cycles rendering engine on the right side. The last thing we need to do before we save and close our scene is to delete the imported Shape_IndexedFaceSet.

Simulation results post processed as they will be rendered

Edit the animate_blender.py script using your preferred editor (Notepad works if you have nothing else). The three highlighted values are mandatory, the rest can be left unless you wanted Shape_IndexedFaceSet to be moved. The paths to the folders are relative to where the script is, so if your folder is in the same folder as the script, the path is just its name as shown in outputFolder. If the folder is in a different folder, you need to include the full path as it's shown in importFolder.

The last thing to do is to run the script. If you are on linux, navigate to the location of the script in the terminal. If you are on windows, open cmd and navigate to the folder there. Once in the folder type :

blender -b filePath --python animate_blender2.9.py
